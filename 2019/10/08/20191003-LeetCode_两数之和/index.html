LeetCode0001两数之和

2. 解题思路

2.1 暴力法

遍历数组，对数组中的每一个元素x，查找是否存在target-x。

class Solution {
    public int[] twoSum(int[] nums, int target) {
        int length = nums.length;
        int right = 0;
        int left = 1;
        for(right = 0; right < length - 1;right++){
            for(left = right + 1; left < length; left++){
                if(nums[right]+nums[left] == target){
                    int[] results = {right,left};
                    return results;
                }
            }
        }
        return null;
    }
}
  • 时间复杂度 O(n^2)
  • 空间复杂度O(1)
  • 执行用时58ms, 22.76%

2.2 使用Hash表

将数组中的数字以数值为key，下标为value加入HashMap；
遍历数组，并在HashMap中查找。

  • 时间复杂度 O(n)
  • 空间复杂度O(n)

2.3 构建Hash表时查找

在将元素x加入HashMap的同时，在现有HashMap中查找是否存在target-x

class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> numsMap = new HashMap<>(); 
        int length = nums.length, i;
        for(i = 0; i < length;i++){
            if(numsMap.containsKey(target - nums[i])){
                return new int[]{numsMap.get(target - nums[i]),i};
            }else{
                numsMap.put(nums[i],i);
            }
        } 
        return null;
    }
}
  • 时间复杂度 O(n)
  • 空间复杂度O(n)
  • 执行用时9ms, 67.24%

2.4 初始化HashMap的长度，预先分配空间，将进一步加快速度

class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> numsMap = new HashMap<>(nums.length); //初始化固定长度，可以加快速度
        int length = nums.length, i;
        for(i = 0; i < length;i++){
            if(numsMap.containsKey(target - nums[i])){
                return new int[]{numsMap.get(target - nums[i]),i};
            }else{
                numsMap.put(nums[i],i);
            }
        } 
        return null;
    }
}
  • 时间复杂度 O(n)
  • 空间复杂度O(n)
  • 执行用时4ms, 93.66%

3. 解题收获

3.1 使用Hash表提高查找速度

Map<Integer, Integer> numsMap = new HashMap<>(nums.length);
HashMap的查找复杂的为O(n)

3.2 预先分配HashMap的空间提高构建速度

HashMap可以预先分配空间

3.3 构建HashMap时顺便查找可以提高速度

考虑是否可以再一次遍历做更多事情

3.4 HashMap与TreeMap的选择

  • HashMap 和TreeMap 都是非线程安全。
  • HashMap 的存储方式为数组，处理冲突的方式为链表，Java8 之后，冲突数超过8，链表转换为红黑树。
  • TreeMap 的存储方式为红黑树，有序；HashMap 无固定顺序。
  • HashMap 适用于在Map中插入、删除和定位元素。
  • TreeMap 适用于按自然顺序或自定义顺序遍历键(key)。
